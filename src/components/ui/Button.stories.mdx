# Button Component

A flexible, accessible button component built with Tailwind CSS and class-variance-authority for type-safe variants.

## Features

- **Multiple Variants**: Default, destructive, outline, secondary, ghost, and link styles
- **Multiple Sizes**: Small, medium (default), large, and icon-only sizes
- **Loading States**: Built-in spinner with proper accessibility
- **Full Accessibility**: ARIA attributes, keyboard navigation, focus management
- **Icon Support**: Proper sizing and spacing for icons
- **Type Safety**: Full TypeScript support with variant props
- **Customizable**: Easy to extend with custom classes

## Import

```tsx
import { Button } from '@/components/ui/button';
```

## Basic Usage

```tsx
// Default button
<Button>Click me</Button>

// With click handler
<Button onClick={() => console.log('Clicked!')}>
  Submit
</Button>
```

## Variants

### Default
The primary button style used for main actions.

```tsx
<Button variant="default">
  Primary Action
</Button>
```

### Destructive
For dangerous or destructive actions like deleting.

```tsx
<Button variant="destructive">
  Delete Item
</Button>
```

### Outline
A subtle button with border, good for secondary actions.

```tsx
<Button variant="outline">
  Secondary Action
</Button>
```

### Secondary
A muted button for less prominent actions.

```tsx
<Button variant="secondary">
  Cancel
</Button>
```

### Ghost
A minimal button that only shows background on hover.

```tsx
<Button variant="ghost">
  Ghost Button
</Button>
```

### Link
Styled like a text link with underline on hover.

```tsx
<Button variant="link">
  Learn More
</Button>
```

## Sizes

### Small
Compact size for tight spaces or less important actions.

```tsx
<Button size="sm">
  Small Button
</Button>
```

### Default (Medium)
Standard size for most use cases.

```tsx
<Button size="default">
  Default Button
</Button>
```

### Large
Prominent size for important actions.

```tsx
<Button size="lg">
  Large Button
</Button>
```

### Icon Only
Square button designed for icons.

```tsx
<Button size="icon">
  ⚙️
</Button>
```

## States

### Disabled
Prevents interaction and provides visual feedback.

```tsx
<Button disabled>
  Disabled Button
</Button>
```

### Loading
Shows spinner and disables interaction.

```tsx
<Button loading>
  Loading...
</Button>
```

### Loading with Custom Text
Maintains button text while showing spinner.

```tsx
<Button loading>
  Saving Changes...
</Button>
```

## Advanced Usage

### Custom Classes
Extend styling with additional Tailwind classes.

```tsx
<Button className="w-full justify-start">
  Full Width Button
</Button>
```

### Form Integration
Use with forms and specify button types.

```tsx
<form>
  <Button type="submit" variant="default">
    Submit Form
  </Button>
  <Button type="button" variant="outline">
    Cancel
  </Button>
</form>
```

### Event Handlers
Handle various user interactions.

```tsx
<Button
  onClick={handleClick}
  onFocus={handleFocus}
  onBlur={handleBlur}
  onMouseEnter={handleMouseEnter}
>
  Interactive Button
</Button>
```

## Accessibility

The Button component follows WCAG accessibility guidelines:

### Keyboard Navigation
- **Enter** and **Space** keys trigger button actions
- Proper focus management with visible focus indicators
- Tab navigation support

### Screen Reader Support
- Proper ARIA attributes for all states
- Loading spinner hidden from screen readers with `aria-hidden="true"`
- Disabled state communicated via `aria-disabled="true"`

### Focus Management
- Clear focus indicators with ring styles
- Focus-visible support for keyboard vs mouse interaction
- Proper focus offset for better visibility

### Loading States
```tsx
const [isLoading, setIsLoading] = useState(false);

const handleSubmit = async () => {
  setIsLoading(true);
  try {
    await submitForm();
  } finally {
    setIsLoading(false);
  }
};

return (
  <Button loading={isLoading} onClick={handleSubmit}>
    {isLoading ? 'Submitting...' : 'Submit'}
  </Button>
);
```

## Design Guidelines

### When to Use Each Variant

- **Default**: Primary actions, form submissions, main CTAs
- **Destructive**: Delete, remove, or other dangerous actions
- **Outline**: Secondary actions, cancel buttons, alternative options
- **Secondary**: Tertiary actions, less important buttons
- **Ghost**: Minimal actions, icon buttons, subtle interactions
- **Link**: Navigation, inline actions, "learn more" links

### Size Guidelines

- **Small**: Use in compact layouts, secondary actions, or when space is limited
- **Default**: Standard size for most buttons in forms and interfaces
- **Large**: Important CTAs, primary actions on landing pages, mobile-first designs
- **Icon**: Tool buttons, action menus, compact icon-only interactions

### Spacing and Layout

- Use consistent spacing between button groups
- Align buttons appropriately (left, center, or right based on context)
- Consider responsive behavior for mobile devices
- Maintain proper spacing from form elements and content

## Technical Details

### CSS Custom Properties
The button uses design tokens from your CSS custom properties:

- `--primary` / `--primary-foreground`
- `--secondary` / `--secondary-foreground`
- `--destructive` / `--destructive-foreground`
- `--accent` / `--accent-foreground`
- `--ring` for focus indicators
- `--radius` for border radius

### Typography
Button typography is configured in Tailwind config:

- `text-button-sm`: 14px with optimized line height
- `text-button-base`: 16px with optimized line height
- `text-button-lg`: 18px with optimized line height

All button text includes `-0.025em` letter spacing for improved readability.

### Performance
The component is optimized for performance:

- Uses `React.forwardRef` for proper ref handling
- Minimal re-renders through stable class generation
- Efficient class merging with `tailwind-merge`
- Lightweight bundle size with tree-shakeable imports

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'destructive' \| 'outline' \| 'secondary' \| 'ghost' \| 'link'` | `'default'` | Button style variant |
| `size` | `'default' \| 'sm' \| 'lg' \| 'icon'` | `'default'` | Button size |
| `loading` | `boolean` | `false` | Shows loading spinner and disables button |
| `disabled` | `boolean` | `false` | Disables button interaction |
| `asChild` | `boolean` | `false` | Render as child element (polymorphic) |
| `className` | `string` | - | Additional CSS classes |
| `onClick` | `(event: MouseEvent) => void` | - | Click event handler |
| `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | Button type attribute |

Plus all standard HTML button attributes (`id`, `aria-*`, `data-*`, etc.).
