# List Component

A flexible, accessible list component with support for various styles, interactive states, and recipe-specific variants. Built with TypeScript and full keyboard accessibility.

## Features

- **Multiple Variants**: Default, bordered, divided, card, inline, and grid layouts
- **Size Options**: Small, medium, and large sizes with adjustable density
- **Interactive States**: Clickable items, checkboxes, loading, and disabled states
- **Icon Support**: Leading icons with proper sizing and spacing
- **Recipe-Specific**: Specialized components for ingredients, instructions, and nutrition
- **Grid Layout**: Responsive grid layouts with configurable columns
- **Full Accessibility**: Proper ARIA attributes, keyboard navigation, focus management
- **Flexible Content**: Support for titles, descriptions, avatars, and custom content

## Checkbox Lists

Create selectable lists with checkboxes.

```tsx
const [selectedItems, setSelectedItems] = useState(new Set());

const handleToggle = (id) => {
  const newSelection = new Set(selectedItems);
  if (newSelection.has(id)) {
    newSelection.delete(id);
  } else {
    newSelection.add(id);
  }
  setSelectedItems(newSelection);
};

return (
  <List variant="divided">
    <CheckboxListItem
      checked={selectedItems.has('notifications')}
      onCheckedChange={() => handleToggle('notifications')}
      label="Enable notifications"
      description="Get notified about new recipes and updates"
    />
    <CheckboxListItem
      checked={selectedItems.has('darkmode')}
      onCheckedChange={() => handleToggle('darkmode')}
      label="Dark mode"
      description="Use dark theme for better night viewing"
    />
    <CheckboxListItem
      checked={selectedItems.has('autosave')}
      onCheckedChange={() => handleToggle('autosave')}
      label="Auto-save recipes"
      description="Automatically save recipe changes"
    />
    <CheckboxListItem
      checked={selectedItems.has('metric')}
      onCheckedChange={() => handleToggle('metric')}
      label="Metric measurements"
      description="Use metric units for ingredients"
      disabled
    />
  </List>
);
```

## Recipe-Specific Lists

Use specialized components for recipe content.

### Ingredient Lists
Display ingredients with quantities and optional checking.

```tsx
const [checkedIngredients, setCheckedIngredients] = useState(new Set());

const ingredients = [
  { id: '1', name: 'All-purpose flour', quantity: '2', unit: 'cups' },
  { id: '2', name: 'Large eggs', quantity: '3', unit: '' },
  { id: '3', name: 'Whole milk', quantity: '1', unit: 'cup' },
  { id: '4', name: 'Unsalted butter', quantity: '4', unit: 'tbsp', optional: true },
  { id: '5', name: 'Salt', quantity: '1', unit: 'tsp' },
  { id: '6', name: 'Baking powder', quantity: '2', unit: 'tsp' }
];

const handleToggleIngredient = (id) => {
  const newChecked = new Set(checkedIngredients);
  if (newChecked.has(id)) {
    newChecked.delete(id);
  } else {
    newChecked.add(id);
  }
  setCheckedIngredients(newChecked);
};

return (
  <RecipeList context="ingredients" variant="divided">
    {ingredients.map(ingredient => (
      <IngredientListItem
        key={ingredient.id}
        ingredient={{
          ...ingredient,
          checked: checkedIngredients.has(ingredient.id)
        }}
        allowChecking
        onToggleCheck={handleToggleIngredient}
        context="shopping"
      />
    ))}
  </RecipeList>
);
```

### Instruction Lists
Display cooking instructions with step numbers and optional completion tracking.

```tsx
const [completedSteps, setCompletedSteps] = useState(new Set());

const instructions = [
  {
    id: '1',
    step: 1,
    text: 'In a large bowl, whisk together flour, salt, and baking powder.'
  },
  {
    id: '2',
    step: 2,
    text: 'In another bowl, beat eggs and gradually add milk.',
    duration: '2 min'
  },
  {
    id: '3',
    step: 3,
    text: 'Melt butter in a skillet over medium heat.',
    temperature: '350°F',
    duration: '1 min'
  },
  {
    id: '4',
    step: 4,
    text: 'Pour wet ingredients into dry ingredients and mix until just combined.',
    duration: '30 sec'
  }
];

const handleToggleStep = (id) => {
  const newCompleted = new Set(completedSteps);
  if (newCompleted.has(id)) {
    newCompleted.delete(id);
  } else {
    newCompleted.add(id);
  }
  setCompletedSteps(newCompleted);
};

return (
  <RecipeList context="instructions" variant="divided">
    {instructions.map(instruction => (
      <InstructionListItem
        key={instruction.id}
        instruction={{
          ...instruction,
          completed: completedSteps.has(instruction.id)
        }}
        allowChecking
        onToggleComplete={handleToggleStep}
        context="cooking"
      />
    ))}
  </RecipeList>
);
```

## Accessibility

The List component follows WCAG accessibility guidelines:

### Keyboard Navigation
- **Arrow keys** navigate between list items
- **Enter** and **Space** activate interactive items
- **Tab** moves focus to next interactive element
- Proper focus management with visible focus indicators

### Screen Reader Support
- Proper list semantics with `<ul>`, `<ol>`, and `<li>` elements
- ARIA attributes for interactive states and descriptions
- Checkbox states communicated via `aria-checked`
- Loading states announced with `aria-busy`
- Disabled states communicated via `aria-disabled`

### Focus Management
- Clear focus indicators with ring styles
- Focus-visible support for keyboard vs mouse interaction
- Logical tab order within complex list items
- Focus restoration after dynamic updates

## Design Guidelines

### When to Use Each Variant
- **Default**: Simple lists, basic content organization
- **Bordered**: Contained lists, forms, settings panels
- **Divided**: Long lists, content separation, readable scanning
- **Card**: Prominent items, dashboard widgets, feature lists
- **Grid**: Categories, dashboards, responsive layouts

### Size Guidelines
- **Small**: Dense interfaces, mobile views, secondary content
- **Medium**: Standard size for most desktop interfaces
- **Large**: Prominent lists, accessibility needs, touch interfaces

### Density Guidelines
- **Compact**: Data-heavy interfaces, tables, admin panels
- **Comfortable**: Standard interfaces, balanced readability
- **Spacious**: Accessibility needs, marketing pages, hero sections

## Props Reference

### List Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'bordered' \| 'divided' \| 'card' \| 'inline' \| 'grid'` | `'default'` | Visual style variant |
| `size` | `'sm' \| 'md' \| 'lg'` | `'md'` | Size of text and spacing |
| `density` | `'compact' \| 'comfortable' \| 'spacious'` | `'comfortable'` | Vertical spacing density |
| `gridCols` | `1 \| 2 \| 3 \| 4` | `1` | Number of grid columns (grid variant only) |
| `ordered` | `boolean` | `false` | Render as ordered list (ol) vs unordered (ul) |
| `children` | `ReactNode` | - | List items |

### ListItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'default' \| 'interactive'` | `'default'` | Item style variant |
| `disabled` | `boolean` | `false` | Whether the item is disabled |
| `loading` | `boolean` | `false` | Whether the item is loading |
| `onSelect` | `() => void` | - | Selection handler (interactive variant) |
| `children` | `ReactNode` | - | Item content |

### CheckboxListItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `checked` | `boolean` | `false` | Whether the checkbox is checked |
| `onCheckedChange` | `(checked: boolean) => void` | - | Change handler |
| `label` | `string` | - | Main label text |
| `description` | `string` | - | Description text |
| `disabled` | `boolean` | `false` | Whether the item is disabled |

### IngredientListItem Props
