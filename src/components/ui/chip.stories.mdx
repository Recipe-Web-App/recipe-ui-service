# Chip Component

Interactive tag component with delete, selection, and click actions. Perfect for managing recipe ingredients, dietary restrictions, and dynamic tags with full accessibility support.

## Features

- **Multiple Variants**: Default, outlined, and filled styles
- **Multiple Sizes**: Small, medium, and large sizes
- **Color Options**: 7 different color variants including primary, success, warning, and destructive
- **Interactive States**: Selection, deletion, click actions, and disabled state
- **Icon Support**: Leading icons and avatar support
- **Full Accessibility**: Proper ARIA attributes, keyboard navigation
- **Recipe-Specific**: Specialized variants for ingredients, dietary restrictions, categories, and tags
- **Group Management**: ChipGroup component for organized layouts
- **Input Component**: ChipInput for dynamic tag creation with validation

## Import

```tsx
import { Chip, ChipGroup, ChipInput, RecipeChip } from '@/components/ui/chip';
```


## Interactive Features

### Deletable Chips

Add a delete button with the `onDelete` prop.

```tsx
<Chip onDelete={() => handleDelete('tomato')}>
  Tomato
</Chip>
<Chip onDelete={() => handleDelete('onion')} variant="outlined">
  Onion
</Chip>
<Chip onDelete={() => handleDelete('garlic')} variant="filled" size="lg">
  Garlic
</Chip>
```

### Selectable Chips

Use the `selected` prop and `onClick` handler for selection.

```tsx
const [selected, setSelected] = useState(['easy']);
const difficulties = ['Easy', 'Medium', 'Hard'];

return (
  <div className="flex gap-3">
    {difficulties.map(difficulty => (
      <Chip
        key={difficulty}
        selected={selected.includes(difficulty.toLowerCase())}
        onClick={() => toggleSelection(difficulty.toLowerCase())}
        variant="outlined"
      >
        {difficulty}
      </Chip>
    ))}
  </div>
);
```

## Chip Input

The `ChipInput` component allows users to dynamically create and manage chips.

### Basic Input

```tsx
const [ingredients, setIngredients] = useState(['Tomato', 'Onion']);

return (
  <ChipInput
    value={ingredients}
    onChange={setIngredients}
    placeholder="Add ingredient..."
    maxChips={8}
  />
);
```

### With Validation

Add custom validation for input values.

```tsx
const [tags, setTags] = useState(['recipe', 'cooking']);

const validateTag = value => {
  if (value.length < 3) return 'Tag must be at least 3 characters';
  if (value.length > 15) return 'Tag must be less than 15 characters';
  if (!/^[a-zA-Z0-9-]+$/.test(value))
    return 'Only letters, numbers, and hyphens allowed';
  return true;
};

return (
  <ChipInput
    value={tags}
    onChange={setTags}
    placeholder="Add tag..."
    validate={validateTag}
    maxChips={10}
  />
);
```

### With Suggestions

Provide suggestions for faster input.

```tsx
const [selected, setSelected] = useState([]);
const suggestions = [
  'Vegetarian',
  'Vegan',
  'Gluten-Free',
  'Dairy-Free',
  'Keto',
  'Low-Carb',
  'Sugar-Free',
  'Paleo',
  'Mediterranean',
];

return (
  <ChipInput
    value={selected}
    onChange={setSelected}
    placeholder="Type to see suggestions..."
    suggestions={suggestions}
  />
);
```

## Advanced Usage

### Filter Interface

Create interactive filter interfaces with chips.

```tsx
const [filters, setFilters] = useState({
  difficulty: ['Easy'],
  time: [],
  category: ['Dinner'],
});

const toggleFilter = (category, value) => {
  setFilters(prev => {
    const currentValues = prev[category];
    const newValues = currentValues.includes(value)
      ? currentValues.filter(v => v !== value)
      : [...currentValues, value];
    return { ...prev, [category]: newValues };
  });
};

return (
  <div className="space-y-4">
    <div>
      <p className="text-muted-foreground mb-2 text-xs">Difficulty</p>
      <div className="flex gap-2">
        {['Easy', 'Medium', 'Hard'].map(level => (
          <Chip
            key={level}
            variant="outlined"
            selected={filters.difficulty.includes(level)}
            onClick={() => toggleFilter('difficulty', level)}
            color={
              level === 'Easy'
                ? 'success'
                : level === 'Medium'
                  ? 'warning'
                  : 'destructive'
            }
          >
            {level}
          </Chip>
        ))}
      </div>
    </div>

    <div>
      <p className="text-muted-foreground mb-2 text-xs">Cook Time</p>
      <div className="flex gap-2">
        {['< 30 min', '30-60 min', '1+ hour'].map(time => (
          <Chip
            key={time}
            variant="outlined"
            selected={filters.time.includes(time)}
            onClick={() => toggleFilter('time', time)}
            icon={<Timer className="h-3 w-3" />}
          >
            {time}
          </Chip>
        ))}
      </div>
    </div>
  </div>
);
```

## Accessibility

The Chip component follows WCAG accessibility guidelines:

### Keyboard Navigation

- **Enter** and **Space** keys trigger chip actions
- **Delete** and **Backspace** keys trigger deletion when focused
- Proper focus management with visible focus indicators
- Tab navigation support for interactive chips

### Screen Reader Support

- Proper ARIA attributes for all states (`aria-selected`, `aria-disabled`)
- Delete buttons have descriptive `aria-label` attributes
- Selected state communicated via `aria-pressed="true"`
- Group semantics with proper ARIA roles

### Focus Management

- Clear focus indicators with ring styles
- Focus-visible support for keyboard vs mouse interaction
- Logical tab order within chip groups
- Focus restoration after deletion

## Design Guidelines

### When to Use Each Variant

- **Default**: General purpose tags, neutral information
- **Outlined**: Secondary information, filter options, selectable states
- **Filled**: Important tags, selected states, prominent categories

### Color Guidelines

- **Primary**: Important actions, main categories
- **Success**: Positive attributes (vegetarian, healthy, easy)
- **Warning**: Cautionary information (medium difficulty, allergens)
- **Destructive**: Negative actions (delete, remove, hard difficulty)
- **Info**: Neutral information (categories, metadata)

### Size Guidelines

- **Small**: Dense layouts, metadata, less important information
- **Medium**: Standard size for most use cases
- **Large**: Important tags, hero sections, prominent categories

## Props Reference

### Chip Props

| Prop       | Type                                                                                         | Default     | Description                          |
| ---------- | -------------------------------------------------------------------------------------------- | ----------- | ------------------------------------ |
| `variant`  | `'default' \| 'outlined' \| 'filled'`                                                        | `'default'` | Visual style variant                 |
| `size`     | `'sm' \| 'md' \| 'lg'`                                                                       | `'md'`      | Size of the chip                     |
| `color`    | `'default' \| 'primary' \| 'secondary' \| 'success' \| 'warning' \| 'destructive' \| 'info'` | `'default'` | Color variant                        |
| `selected` | `boolean`                                                                                    | `false`     | Whether the chip is selected         |
| `disabled` | `boolean`                                                                                    | `false`     | Whether the chip is disabled         |
| `icon`     | `ReactNode`                                                                                  | -           | Leading icon element                 |
| `avatar`   | `ReactNode`                                                                                  | -           | Avatar element (replaces icon)       |
| `onDelete` | `() => void`                                                                                 | -           | Delete handler (shows delete button) |
| `onClick`  | `() => void`                                                                                 | -           | Click handler                        |
| `children` | `ReactNode`                                                                                  | -           | Chip content                         |

### ChipGroup Props

| Prop         | Type                             | Default    | Description                              |
| ------------ | -------------------------------- | ---------- | ---------------------------------------- |
| `spacing`    | `'tight' \| 'normal' \| 'loose'` | `'normal'` | Spacing between chips                    |
| `maxDisplay` | `number`                         | -          | Maximum chips to display before overflow |
| `children`   | `ReactNode`                      | -          | Chip elements                            |

### ChipInput Props

| Prop          | Type                                   | Default | Description                     |
| ------------- | -------------------------------------- | ------- | ------------------------------- |
| `value`       | `string[]`                             | -       | Array of current values         |
| `onChange`    | `(values: string[]) => void`           | -       | Change handler                  |
| `placeholder` | `string`                               | -       | Input placeholder text          |
| `maxChips`    | `number`                               | -       | Maximum number of chips allowed |
| `validate`    | `(value: string) => boolean \| string` | -       | Validation function             |
| `suggestions` | `string[]`                             | -       | Array of suggestion values      |

### RecipeChip Props

| Prop       | Type                                               | Default | Description                  |
| ---------- | -------------------------------------------------- | ------- | ---------------------------- |
| `context`  | `'ingredient' \| 'dietary' \| 'category' \| 'tag'` | -       | Recipe-specific context      |
| `selected` | `boolean`                                          | `false` | Whether the chip is selected |
| `onDelete` | `() => void`                                       | -       | Delete handler               |
| `children` | `ReactNode`                                        | -       | Chip content                 |

Plus all standard chip props and HTML attributes.
