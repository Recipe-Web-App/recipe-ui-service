# Disclosure Component

A simple toggle for showing/hiding a single section of content. Perfect for FAQ items, recipe notes, cooking tips, and other collapsible content.

## Features

- **Simple API**: Easy to use compound component pattern
- **Smooth Animations**: CSS transitions for expand/collapse
- **Multiple Variants**: Default, outlined, minimal, elevated, and card styles
- **Recipe-Specific**: Pre-configured components for cooking use cases
- **Full Accessibility**: ARIA attributes, keyboard navigation, screen reader support
- **Controlled/Uncontrolled**: Support both usage patterns
- **Icon Support**: Customizable icons with rotation animations
- **Type Safety**: Full TypeScript support with variant props

## Import

```tsx
import {
  Disclosure,
  DisclosureTrigger,
  DisclosureContent,
  RecipeDisclosure,
  KitchenTipsDisclosure,
  FAQDisclosure,
  IngredientNotesDisclosure,
  DisclosureGroup,
} from '@/components/ui/disclosure';
```

## Basic Usage

```tsx
// Simple question and answer
<Disclosure>
  <DisclosureTrigger>What ingredients do I need?</DisclosureTrigger>
  <DisclosureContent>
    <p>This recipe requires: flour, eggs, milk, butter, salt, and sugar.</p>
  </DisclosureContent>
</Disclosure>

// Open by default
<Disclosure defaultOpen>
  <DisclosureTrigger>Important note</DisclosureTrigger>
  <DisclosureContent>
    <p>This content is visible by default.</p>
  </DisclosureContent>
</Disclosure>

// Controlled usage
<Disclosure open={isOpen} onOpenChange={setIsOpen}>
  <DisclosureTrigger>Controlled disclosure</DisclosureTrigger>
  <DisclosureContent>
    <p>This disclosure is controlled by external state.</p>
  </DisclosureContent>
</Disclosure>
```

## Accessibility

The Disclosure component includes comprehensive accessibility features:

- **ARIA Attributes**: Proper `aria-expanded`, `aria-controls`, and `aria-labelledby`
- **Keyboard Navigation**: Full support for Tab, Enter, and Space keys
- **Screen Reader Support**: Content regions properly labeled
- **Focus Management**: Logical focus flow and visible focus indicators

### Keyboard Navigation

- **Tab**: Focus the trigger button
- **Enter/Space**: Toggle open/closed state
- **Tab (when open)**: Move focus to content area

### Screen Reader Behavior

Screen readers will announce:

- Button role and expanded state
- Content region when opened
- Proper labeling relationships between trigger and content

## Best Practices

### Content Structure

- Keep trigger text concise and descriptive
- Use semantic HTML within content areas
- Provide clear visual hierarchy

### Performance

- Use `forceMount` sparingly (only when needed for SEO)
- Prefer CSS transitions over JavaScript animations
- Consider content length for smooth animations

### Recipe Context Usage

- Use `RecipeDisclosure` for cooking-related content
- Choose appropriate context (tips, notes, substitutions, etc.)
- Add badges for important information

### Grouping

- Use `DisclosureGroup` for related sets of disclosures
- Choose appropriate spacing (tight, normal, loose)
- Consider allowing multiple open disclosures vs single
