# EmptyState Component

A flexible, accessible empty state component for when lists/grids have no content (no recipes, empty search results, etc.).

## Features

- **Multiple Variants**: Default, search, minimal, illustrated, and error states
- **Multiple Sizes**: Small, medium (default), and large for different contexts
- **Compound Components**: Flexible composition with Icon, Title, Description, and Actions
- **Full Accessibility**: ARIA attributes, semantic HTML, keyboard navigation
- **Responsive Design**: Mobile-first approach with adaptive layouts
- **Type Safety**: Full TypeScript support with variant props
- **Customizable**: Easy to extend with custom classes and content


## Accessibility

The EmptyState component follows WCAG accessibility guidelines:

### Semantic Structure
- Uses proper heading hierarchy with `h3` for titles
- Includes `role="status"` for screen reader announcements
- Provides `aria-live="polite"` for dynamic content updates

### Screen Reader Support
- Icons are hidden from screen readers with `aria-hidden="true"`
- Descriptive text provides context for empty states
- Action buttons have clear, descriptive labels

### Keyboard Navigation
- All interactive elements are keyboard accessible
- Proper tab order through action buttons
- Focus indicators for keyboard users

```tsx
// Accessible empty state with custom ARIA labels
<EmptyState aria-label="Empty recipe collection">

    <Button aria-label="Clear search filters">Clear Filters</Button>
    <Button aria-label="Browse all available recipes">Browse All</Button>
  </EmptyStateActions>
</EmptyState>
```

## Design Guidelines

### When to Use Each Variant

- **Default**: General empty states, main content areas, recipe collections
- **Search**: Empty search results, filtered lists with no matches
- **Minimal**: Secondary content areas, subtle empty states, loading states
- **Illustrated**: Landing pages, onboarding flows, prominent empty states
- **Error**: Failed operations, connection errors, import failures

### Size Guidelines

- **Small**: Sidebars, cards, modal content, secondary sections
- **Medium**: Standard content areas, search results, list views
- **Large**: Landing pages, main dashboards, prominent features

### Content Guidelines

- **Titles**: Keep concise (2-5 words), describe the empty state clearly
- **Descriptions**: Explain why it's empty and suggest next steps (1-2 sentences)
- **Actions**: Use action-oriented language, limit to 1-3 primary actions
- **Icons**: Choose relevant, recognizable icons that support the message

### Spacing and Layout

- Use consistent spacing between components via size variants
- Align content centrally for better visual balance
- Consider responsive behavior for mobile devices
- Maintain proper spacing from surrounding content

## Technical Details

### CSS Custom Properties
The component uses design tokens from your CSS custom properties:

- `--card` / `--card-foreground` for default variant
- `--muted` / `--muted-foreground` for search and minimal variants
- `--destructive` / `--destructive-foreground` for error variant
- `--primary` for illustrated variant accents
- `--border` for border colors and styling

### Performance
The component is optimized for performance:

- Uses `React.forwardRef` for proper ref handling
- Compound components enable tree-shaking
- Efficient class merging with `tailwind-merge`
- Minimal re-renders through stable class generation

### Responsive Design
Built with mobile-first approach:

- Flexible layouts that adapt to container width
- Responsive text sizing and spacing
- Touch-friendly action button sizing
- Proper spacing on smaller screens
