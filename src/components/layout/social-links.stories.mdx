## Accessibility Features

- **ARIA Labels**: Each link includes descriptive aria-label text
- **Screen Reader Support**: Clear context about destination and purpose
- **Keyboard Navigation**: All links are keyboard accessible with proper focus indicators
- **External Link Safety**: All links include `rel="noopener noreferrer"` for security
- **Icon Accessibility**: Icons are marked as decorative with `aria-hidden="true"`

## Design Guidelines

### Icon Sizes

- **Small (sm)**: 16px - Best for footers and compact areas
- **Medium (md)**: 20px - Good for sidebars and general use
- **Large (lg)**: 24px - Ideal for hero sections and prominent placement

### Spacing

- Consistent 16px gap between icons
- Adequate touch targets for mobile devices
- Proper alignment with surrounding content

### Hover Effects

- Smooth scale animation (1.1x on hover)
- Color transitions for visual feedback
- Maintains accessibility during animations

## Implementation Notes

- The component uses a safe object access pattern to prevent injection attacks
- All links automatically open in new tabs for external platforms
- Default links include GitHub and Twitter if no custom links are provided
- Icons are sourced from the Lucide React library for consistency
- The component gracefully handles invalid platform names by skipping them
- TypeScript interfaces ensure type safety for platform names and configurations
